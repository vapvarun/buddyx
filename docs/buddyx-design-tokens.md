# BuddyX Design Tokens

Stable CSS variable contract for theme stylesheets, child themes, and
plugin extensions. Generated by `inc/Tokens/Component.php` from customizer
theme_mods on every front-end render.

**Naming convention:** `--bx-{kind}-{role}-{state?}`
- `kind` = `color` or `radius`
- `role` = semantic identifier (accent, fg, bg, link, button, header, …)
- `state` = optional suffix (`-hover`, `-active`, `-fg`, `-bg`)

## Backward-compatibility aliases (5.0.3 names)

Every token above also emits its legacy alias used by the 5.0.3 stylesheet:

```css
:root {
  /* New token */
  --bx-color-accent: #ef5455;
  /* Legacy alias (deprecated, will be removed in 5.3.0) */
  --color-theme-primary: #ef5455;
}
```

Full alias map lives in `inc/Tokens/Component.php::$simple_color_tokens`
(plus `$typography_color_tokens` and `$dimension_tokens`). Third-party CSS
hooked to the legacy names continues to work unchanged through the 5.2.x
release line. We recommend migrating consumers to `--bx-` tokens before
5.3.0 ships.

## Color modes (5.2.0+)

The token set is designed to support light/dark mode swaps via a single
attribute on `<html>`:

```css
:root              { --bx-color-bg: #ffffff; --bx-color-fg: #111; … }
[data-bx-mode=dark]:root { --bx-color-bg: #0a0a0a; --bx-color-fg: #f5f5f5; … }
```

The customizer field `site_color_mode` (auto/light/dark) controls which
attribute value is set on `<html>`. Theme stylesheets only ever reference
the token name — never light or dark hex values directly — so a mode
swap is one CSS variable change, not a stylesheet swap.

## How to consume in a stylesheet

```css
.my-button {
  background: var(--bx-color-button-bg);
  color: var(--bx-color-button-fg);
  border-radius: var(--bx-radius-button);
  transition: background 0.15s;
}
.my-button:hover {
  background: var(--bx-color-button-bg-hover);
  color: var(--bx-color-button-fg-hover);
}
```

## How to consume in PHP

Direct read of the underlying setting:

```php
$accent = get_theme_mod( 'site_primary_color', '#ef5455' );
```

Or iterate the token map (rare — usually CSS is the right consumer):

```php
$tokens = \BuddyX\Buddyx\Tokens\Component::get_simple_color_tokens();
foreach ( $tokens as $mod_key => $cfg ) {
    echo $cfg['token']; // e.g. '--bx-color-accent'
    echo $cfg['aliases'][0]; // e.g. '--color-theme-primary'
}
```
